## FAQ（常见问题）

在使用码镜过程中，如遇到任何问题，欢迎及时反馈。我们会将问题纳入常见问题列表持续跟踪，并尽快推进修复与优化。

| 问题id | 问题描述 | 状态 | 说明 |
| --- | --- | --- | --- |
| 1 | Mac 原生动态壁纸介绍 | 说明项 | 从实现机制看，macOS 严格意义上并不是在桌面态持续播放“原生动态壁纸”，而是“动态屏保 + 静态壁纸”的组合方案。早期动态屏保素材形态较多，除视频外也出现过 HEIC 等格式；目前主流形态已基本收敛为视频屏保。其典型工作流程是：系统进入锁屏或屏保状态后开始播放视频；当用户回到桌面时，系统通过平滑过渡将视频停在某一帧，并将该帧设为静态桌面壁纸；再次进入锁屏或屏保时，再从该帧附近继续播放，从而形成连贯、自然的视觉体验。 |
| 2 | 能否实现 Mac 原生的效果？ | 说明项 | 若希望将自定义视频做成与 macOS 原生机制完全一致的效果，正规路径通常不可行。网上常见的做法是替换系统目录中的原生视频资源，但该方案在不同系统版本上的表现差异很大：有的版本可能可用，有的版本可能需要关闭系统完整性保护（SIP）后才能操作。无论成功率还是系统安全性，这种方式都存在明显风险，因此不建议作为常规方案。 |
| 3 | 码镜当前存在的明显缺陷是什么？ | 说明项 | 受系统接口能力限制，码镜的动态屏保无法完全复现原生壁纸那种“屏保→桌面”丝滑衔接体验，整体更接近“各自独立播放”的模式。你失去的是原生级过渡动画，但获得的是可持续播放、可自定义的动态桌面能力，属于体验与可控性的取舍。另一个限制是：在用户主动锁屏场景下，码镜屏保通常无法被系统直接拉起；需要通过“自动启动屏保”或配置左下角热区触发屏保后才能生效，否则锁屏界面仍为静态画面。同时，屏保退出回到桌面时的过渡也不会像原生方案那样平滑。 |
| 4 | 码镜支持那些类型的壁纸？ | 说明项 | 码镜当前支持主流视频类壁纸，具体格式为 `mp4` 与 `mov`，其他视频格式暂未适配。同时已支持 2D 场景壁纸，3D 场景壁纸仍在后续评估与规划中。场景类壁纸（尤其 2D/3D）相较纯视频对设备性能要求更高：在当前主流 Mac 设备上，2D 场景已可稳定使用，通常不会明显影响整机性能；3D 场景仍处于调研阶段。相比视频壁纸，场景类壁纸在交互性和自定义能力上更有优势，更适合按个人创意进行二次开发。 |
| 5 | 能否对码镜运行时性能做简单分析？| 说明项 | 软件稳定性与性能始终是一个关键且无法回避的话题。为此我们专门进行了性能测试，详细结果与分析请参阅「性能测试」章节。 |
| 6 | 码镜为什么要提供登录选项？是否会获取用户隐私？ | 说明项 | 码镜接入服务器的核心目的是为用户提供壁纸素材下载与分发能力；若完全离线，将无法获取在线素材。当前后端架构基于 Supabase 与 Cloudflare 搭建，在稳定性与安全性方面具备完善保障。登录环节采用苹果账号鉴权，码镜不会获取用户敏感隐私信息，可放心使用。引入登录机制一方面用于限制匿名请求、降低壁纸数据被批量抓取及恶意攻击服务端的风险；另一方面也为后续 UGC（用户上传作品）能力提供账号体系与权限基础。 |
| 7 | 安装码镜屏幕保护程序后，点击设置屏保的选项按钮不生效 | 阻塞 | 经过多轮复测，该问题可以稳定复现：首次打开选项面板后可正常关闭，但第二次通常无法再次打开。当前判断可能与 macOS 系统层行为有关，短期内暂无稳定修复方案。临时规避方式：退出“设置”后重新进入，再次打开即可恢复。 |
